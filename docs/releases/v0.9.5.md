# Release v0.9.5 — MCP Server Integration

**Release Date:** February 20, 2026  
**Previous Version:** v0.9.0

---

## Overview

v0.9.5 introduces a Model Context Protocol (MCP) server that wraps all existing CLI commands as AI-agent-callable tools. This enables Claude, GPT, and other AI agents to orchestrate content pipeline workflows programmatically without any new pipeline logic — all tools delegate to existing code.

---

## What's New

### MCP Server (`mcp_server/`)

- FastMCP-based server with stdio (for Kiro/Claude Desktop) and SSE transport
- 6 registered tools wrapping existing pipeline commands:
  - `extract` — Extract audio and metadata from video URLs or local files
  - `transcribe` — Transcribe audio to structured transcript
  - `enrich` — Generate semantic enrichment from transcript using LLM
  - `format_content` — Format enriched content into publishing-ready output
  - `validate` — Validate pipeline artifacts against schemas and platform limits
  - `run_pipeline` — Convenience tool chaining extract → transcribe → enrich → format
- YAML-based server configuration with environment variable overrides
- Structured JSON responses with success/error status for all tools

### Server Configuration

```json
{
    "mcpServers": {
        "content-pipeline": {
            "command": "python",
            "args": ["-m", "mcp_server"],
            "env": {
                "CONTENT_PIPELINE_CONFIG": ".content-pipeline/config.yaml"
            }
        }
    }
}
```

---

## Usage

```bash
# Start server (stdio mode for Kiro/Claude Desktop)
python -m mcp_server

# Start server (SSE mode for remote access)
python -m mcp_server --transport sse --port 8080
```
