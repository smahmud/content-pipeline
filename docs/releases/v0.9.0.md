# Release v0.9.0 — Validation & Quality Gates

**Release Date:** February 20, 2026  
**Previous Version:** v0.8.7

---

## Overview

v0.9.0 introduces the `validate` command for verifying pipeline artifacts against their Pydantic schemas and platform requirements. It reuses existing validators (InputValidator, PlatformValidator) and wraps them in a unified CLI with batch support, JSON reports, and strict mode for CI/CD integration.

---

## What's New

### Validate Command

New CLI command `content-pipeline validate` with full support for:

- **Schema Validation** — Validates TranscriptV1, EnrichmentV1, and FormatV1 artifacts against their Pydantic schemas with field-level error details
- **Auto-Detection** — Automatically detects schema type from file content (enrichment_version, format_version, metadata.schema_version)
- **Platform Validation** — Checks content against platform-specific limits (character counts, hashtags, links) for all 11 supported platforms
- **Batch Validation** — Validates multiple files with glob patterns and generates consolidated reports
- **JSON Reports** — Machine-readable validation reports for CI/CD pipelines (`--report`)
- **Per-File Reports** — Individual report files for batch runs (`--output-dir`)
- **Strict Mode** — Optional `--strict` flag that treats warnings as errors
- **Cross-Reference Validation** — Checks source file existence and enrichment type consistency

### Validation Engine (`pipeline/validation/`)

New module with:

- `ValidationEngine` — Central orchestrator with auto-detect, batch, and strict mode
- `SchemaValidator` — Pydantic schema wrappers returning `ValidationIssue` lists
- `CrossReferenceValidator` — Validates relationships between pipeline artifacts
- `ValidationReport` / `ValidationIssue` — Structured report dataclasses with JSON and human-readable output

## Usage Examples

```bash
# Single file validation (auto-detect schema)
content-pipeline validate --input enriched.json

# Explicit schema type
content-pipeline validate --input enriched.json --schema enrichment

# Platform validation
content-pipeline validate --input enriched.json --platform twitter

# Strict mode
content-pipeline validate --input enriched.json --strict

# Batch with consolidated report
content-pipeline validate --batch "outputs/*.json" --report report.json

# Batch with per-file reports
content-pipeline validate --batch "outputs/*.json" --output-dir ./reports
```
