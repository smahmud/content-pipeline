# Release v0.10.0 — REST API

**Release Date:** February 20, 2026  
**Previous Version:** v0.9.5

---

## Overview

v0.10.0 adds a FastAPI-based REST API exposing all pipeline operations via HTTP endpoints. This provides a universal HTTP interface for web apps, mobile apps, and third-party integrations, complementing the CLI and MCP server interfaces.

---

## What's New

### REST API (`api/`)

- FastAPI application at `/api/v1/` with auto-generated Swagger docs at `/docs`
- 7 pipeline endpoints: `extract`, `transcribe`, `enrich`, `format`, `validate`, `pipeline`, plus health/version/tools
- API key authentication via `X-API-Key` header (optional, disabled by default)
- CORS middleware for cross-origin web client access
- Pydantic request/response models for all endpoints
- Reuses MCP tool wrappers — zero duplicated pipeline logic

### Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/v1/health` | Health check |
| GET | `/api/v1/version` | Version info |
| GET | `/api/v1/tools` | List available tools |
| POST | `/api/v1/extract` | Extract audio from URL/file |
| POST | `/api/v1/transcribe` | Transcribe audio |
| POST | `/api/v1/enrich` | Enrich transcript with LLM |
| POST | `/api/v1/format` | Format enriched content |
| POST | `/api/v1/validate` | Validate pipeline artifacts |
| POST | `/api/v1/pipeline` | Run full pipeline |

## Usage

```bash
# Start API server
uvicorn api.app:app --reload --port 8000

# Health check
curl http://localhost:8000/api/v1/health

# Validate a file
curl -X POST http://localhost:8000/api/v1/validate \
  -H "Content-Type: application/json" \
  -d '{"input_path": "./enriched.json"}'

# Swagger docs
open http://localhost:8000/docs
```
