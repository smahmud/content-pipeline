# Release v0.8.0 - Content Formatting

**Release Date:** February 18, 2026

## Overview

v0.8.0 introduces the `format` command, completing the content pipeline workflow from extraction to publishing-ready formats. This release enables transformation of enriched content into 16 different output formats using a hybrid template + LLM architecture.

## What's New

### Format Command

The new `format` command transforms enriched JSON content into various publishing formats:

```bash
# Generate a blog post
content-pipeline format --input enriched.json --type blog --output blog.md

# Generate with LLM enhancement
content-pipeline format --input enriched.json --type tweet --provider local-ollama

# Generate a bundle of formats
content-pipeline format --input enriched.json --bundle blog-launch --output-dir ./output/
```

### 16 Output Format Types

| Category | Formats |
|----------|---------|
| Publishing | blog, newsletter, seo |
| Social Media | tweet, linkedin, youtube |
| Video | chapters, video-script, tiktok-script |
| Audio | podcast-notes, transcript-clean |
| Notes | notion, obsidian, slides |
| Business | meeting-minutes, quote-cards |

### Pre-configured Bundles

Generate multiple formats at once with named bundles:

- `blog-launch` - Blog + tweet + linkedin + seo
- `video-launch` - YouTube + chapters + tweet + blog
- `podcast` - Podcast notes + newsletter + tweet + transcript
- `social-only` - Tweet + linkedin
- `full-repurpose` - All major formats
- `notes-package` - Notion + obsidian + slides

### Hybrid Architecture

The formatter uses a two-stage approach:
1. **Jinja2 Templates** - Provide consistent structure for each format
2. **LLM Enhancement** - Polish prose and adapt tone (optional)

This ensures reliable output even without LLM access, with graceful degradation.

### Style Profiles

Customize output with style profiles for consistent brand voice:

```bash
content-pipeline format --input enriched.json --type linkedin --style-profile ./profiles/professional.md
```

## Usage Examples

### Single Format Generation

```bash
# Template-only (no LLM)
content-pipeline format --input enriched.json --type blog --no-llm

# With LLM enhancement
content-pipeline format --input enriched.json --type blog --provider local-ollama

# With style profile
content-pipeline format --input enriched.json --type linkedin --style-profile ./my-profile.md
```

### Bundle Generation

```bash
# List available bundles
content-pipeline format --list-bundles

# Generate blog-launch bundle
content-pipeline format --input enriched.json --bundle blog-launch --output-dir ./output/
```

### Batch Processing

```bash
# Process multiple files
content-pipeline format --batch "*.enriched.json" --type blog --output-dir ./blogs/
```

### Cost Control

```bash
# Dry run to estimate costs
content-pipeline format --input enriched.json --type blog --dry-run

# Set cost limit
content-pipeline format --input enriched.json --type blog --max-cost 0.50
```

## Complete Workflow

The content pipeline now supports the full workflow:

```
extract → transcribe → enrich → format
```

1. `extract` - Get audio from video source
2. `transcribe` - Convert audio to text
3. `enrich` - Add semantic metadata (summaries, tags, chapters, highlights)
4. `format` - Transform to publishing formats

## Technical Details

- Uses `pipeline/llm/` infrastructure from v0.7.5
- 228 tests (25 integration + 203 property tests)
- Supports all LLM providers: local-ollama, cloud-openai, cloud-anthropic, cloud-aws-bedrock
